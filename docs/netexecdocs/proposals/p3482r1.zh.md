# P3482R1：基于 IETF TAPS 的 C++ 网络 API

> 原文：[https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2025/p3482r1.html](https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2025/p3482r1.html)

## 摘要

本文提议修改 [p2762] 提出的 API，以支持 IETF TAPS，作为 C++ 标准网络库的基础。

## 修订历史

### r0 → r1

- 修正排版问题。
- 新增 `std::octet`。
- 将由互联网协议 RFC 定义的、精确 8 位的类型改为使用 `std::octet`。
- 强化对 [p1860] 的引用，强调"默认安全连接"的要求。

### r0

[p3482R0] 是本文的首次发布版本。

## 背景

- **[p2762]**：提议修改 Networking TS，采用 Sender/Receiver 模型（目标 C++26）。
- **[p3185]**：提议基于 IETF TAPS 的 C++ 网络方向。TAPS 获取连接的过程与 Berkeley Sockets 或 Networking TS、ASIO 等基于 socket 的模型有本质不同。
- **[p1861]**：基于 Apple Network.Framework [ANF] 提出的接口草案，与 TAPS 有设计渊源。

## TAPS 方法

IETF TAPS 使用**基于属性的方法**描述连接需求。TAPS 基础设施根据这些属性选择一个或多个满足条件的候选方案，然后请求建立符合这些属性的连接。

### 预连接（Preconnection）

创建连接从**预连接**开始。TAPS 用四组属性定义预连接：

- 本地端点（Local Endpoint）
- 远程端点（Remote Endpoint）
- 传输（Transport）
- 安全（Security）

### 端点（Endpoints）

端点定义连接的起点和终点，分为本地和远程。属性包括：

- `hostname`，例如 `"nyarlathotep.example.org"`
- `interface`，例如 `"en0"`
- `service`，例如 `"https"`
- `multicast_group`，例如 `"224.0.0.252"` 或 `"ff02::114"`
- `hop_limit`，例如 `2`

### 传输（Transports）

传输由一组底层基础设施应满足的需求定义。大多数传输需求表示为**偏好**：

- `Require`（必须）
- `Prefer`（优先）
- `None`（无要求）
- `Avoid`（避免）
- `Prohibit`（禁止）

传输属性包括：

- `interface`：多值（接口，偏好）元组集合
- `reliability`：偏好
- `preserve_msg_boundaries`：偏好
- `per_msg_reliability`：偏好
- `preserve_order`：偏好
- `zero_rtt_msg`：偏好
- `multistreaming`：偏好
- `full_checksum_send` / `full_checksum_recv`：偏好
- `congestion_control`：偏好
- `keep_alive`：偏好
- `use_temp_local_address`：偏好
- `multipath`：枚举 `{ disabled, active, passive }`
- `advertises_alt_addr`：偏好
- `direction`：枚举 `{ both, send, recv }`
- `soft_error_notify`：偏好
- `active_read_before_send`：偏好

### 安全（Security）

安全属性包括：

- `allowed_security_protocols`：协议标识符序列
- `server_cert` / `client_cert` / `pinned_server_cert`：证书序列
- `alpn`：应用层协议协商值序列
- `supported_group`：组标识符序列
- `ciphersuite`：密码套件序列
- `signature_algorithm`：签名算法序列
- `max_cached_session`：整数
- `cached_session_lifetime`：时长
- `preshared_key`：密钥材料
- `trust_verification_handler`：序列发送器
- `challenge_handler`：序列发送器

### 预连接与连接

端点、传输和安全属性集用于创建预连接。预连接用于建立连接。TAPS 定义三种方法：

- **Initiate** → Connection，**主动打开**（可命名为 `client_connection`）
- **Listen** → Connection，**被动打开**（可命名为 `server_connection`）
- **Rendezvous** → Connection，**点对点连接**（可命名为 `peer_to_peer_connection`）

预连接可以被修改，但修改只影响后续连接。

预连接还可以执行端点解析：

- **Resolve** → `(local_endpoint[], remote_endpoint[])`

### 面向消息

TAPS 明确是**面向消息**的。这与 Berkeley Sockets 或 Networking TS 不同，后者只是随着数据到达交付缓冲区，由应用自己处理分帧。

TAPS 引入 **Framer（帧解析器/分帧器）** 的概念：从一个或多个接收到的数据块中产生完整消息（或错误）。Framer 是建立连接时的可选参数，是事件驱动的。

- `default_framer`：行为类似 Berkeley socket，直接返回当前收到的字节。
- `http_framer`：从字节流中解析 HTTP 头和消息边界。
- 用户自定义 framer：可用 Thrift 序列化/反序列化，产出强类型领域对象。

#### 消息上下文（Message Contexts）

发送和接收数据时可提供 `MessageContext`。操作完成后，事件处理程序会收到与该事件关联的 `MessageContext`。用途包括：

- 向 framer 传递控制分帧行为的属性
- 与最终的 `set_value()` 处理程序通信
- 关联多个部分发送/接收操作
- 包含连接信息（接口、远程端点等）
- Framer 可扩展消息上下文属性，例如 HTTP 头

## 设计讨论

总体方向：在合理的地方采用 TAPS 概念（属性描述、面向消息、用户可扩展的分帧和消息类型），其余部分尽量采用并修改 [p2762]。

例如，[p2762] 定义了 socket 类型和一组 CPO。本文提议 `connect()`、`listen()`、`rendezvous()` 返回 `connection`，然后 `async_send()`、`async_receive()` 等 CPO 在 `connection` 上操作。

[p1860] 主张"C++ 网络必须默认安全"。使用最小默认安全属性时，实现**必须**创建"安全"传输连接。

[p3477] 讨论字节定义。但网络需要更精确：IP 协议基于**八位组（octet）**，精确 8 位。本文提议定义 `std::net::octet` 为精确 8 位的类型。

TAPS 期望连接是单一对象类型，暴露所有操作。**本文提议每种连接是不同类型**，只暴露该传输类型支持的操作。各种连接操作作为 CPO，基于 preconnection 参数类型定制，允许支持非 TCP/IP 传输。

标准库提供默认传输实现，TCP/IP 相关类型放在 `net::ip` 命名空间。

预连接由描述端点、传输和安全需求的属性集构造。P3325 描述了一种为 `std::execution` 环境构造和查询强类型属性的机制。让 preconnection 构造函数是模板，并要求参数类型符合 `queryable` concept，用户可提供任何满足约束的类型。相关探索中的 `runtime_env` 也符合 `queryable` concept。

属性集合应是开放的。本文提议 `connect()`、`listen()`、`rendezvous()` 表示为 CPO，允许定制如何处理 preconnection 属性。

连接协商、建立、分帧、消息收发都是异步的，应使用 sender/receiver 模型。

[p2762] 的若干操作被 TAPS 设计取代：

- `async_accept()` 被 `async_listen()` 取代
- `async_connect()` 被 `async_initiate()` 取代
- `async_resolve_address()` / `async_resolve_name()` 被 `async_resolve()` 取代

TAPS 是面向消息的。`async_send()` 发送完整消息，编码方式由 Framer 决定。`async_receive()` 接收完整消息，流/数据报无区别，因此移除 `async_receive_from()`。TAPS 也支持部分接收，类似 `async_read_some()`。

## 提议 API

以下类型主要来自 [p3325]，位于 `std::execution` 命名空间。

### 6.1 `std::execution::property_key_list`

属性键列表类型。

```cpp
namespace std::execution {
    template<typename... Ts>
    class property_key_list { };
}
```

### 6.2 `std::execution::is_property_key_list_v`

判断类型是否为 `property_key_list`。

```cpp
namespace std::execution {
    template<typename T>
    inline constexpr bool is_property_key_list_v = false;

    template<typename... Ts>
    inline constexpr bool is_property_key_list_v<property_key_list<Ts...>> = true;
}
```

### 6.3 `std::execution::queryable`

判断类型是否为可查询环境。

```cpp
namespace std::execution {
    template<typename T>
    concept queryable =
        requires {
            typename T::property_keys;
            requires is_property_key_list_v<typename T::property_keys>;
        };
}
```

### 6.4 `std::execution::has_query`

判断环境是否支持某查询。

```cpp
namespace std::execution {
    template<typename E, typename Q>
    concept has_query =
        requires (E const& env) { env.query(Q{ }); };
}
```

### 6.5 `std::execution::has_query_default`

判断某查询是否有默认值。

```cpp
namespace std::execution {
    template<typename Q>
    concept has_query_default =
        requires { Q::default_value(); };
}
```

### 6.6 `std::execution::property`

判断类型是否为属性（空类型、可默认构造）。

```cpp
namespace std::execution {
    template<typename T>
    concept property =
        std::is_empty_v<T> && std::default_initializable<T>;
}
```

### 6.7 `std::execution::has_property`

判断可查询对象是否有某属性。

```cpp
namespace std::execution {
    template<typename Q, typename P>
    concept has_property =
        queryable<Q> && property<P> && has_query<Q, P>;
}
```

### 6.8 `std::execution::has_try_query`

判断环境是否支持 `try_query`。

```cpp
namespace std::execution {
    template<typename E, typename Q>
    concept has_try_query =
        requires (E const& env) { env.try_query(Q{ }); };
}
```

### 6.9 `std::execution::maybe_has_property`

判断可查询对象可能拥有某属性。

```cpp
namespace std::execution {
    template<typename Q, typename P>
    concept maybe_has_property =
        queryable<Q> && property<P> && has_try_query<Q, P>;
}
```

### 6.10 `std::execution::runtime_property`

判断类型是否为运行时类型擦除属性。

```cpp
namespace std::execution {
    template<typename T>
    concept runtime_property =
        property<T> && requires { typename T::type_erased_type; };
}
```

### 6.11 `std::execution::runtime_env`

运行时类型擦除的可查询环境。

```cpp
namespace std::execution {
    class runtime_env {
    public:
        runtime_env() noexcept = default;
        operator runtime_env_ref() const noexcept;

        template<runtime_property P, typename Tp>
            requires std::constructible_from<typename P::type_erased_type, Tp>
        void set(P prop, Tp&& init);

        template<runtime_property P>
        void unset(P) noexcept;

        template<runtime_property P>
        std::optional<typename P::type_erased_type> try_query(P prop) const
            noexcept(std::is_nothrow_copy_constructible_v<typename P::type_erased_type>);
    };
}
```

#### 6.13.1 `std::net::ip::address_v4`

#### 6.14.4 `std::net::interface_preference`

```cpp
        interface_preference() = default;

        const_iterator begin() const noexcept;

        void unset(interface iface) noexcept;
```

### 6.16 预连接

#### 6.16.1 `net::preconnection`

```cpp
namespace std::net {
    class preconnection {
    public:
        template<execution::queryable LocalEndpoint,
                 execution::queryable RemoteEndpoint,
                 execution::queryable TransportProps,
                 execution::queryable SecurityProps>
        preconnection(LocalEndpoint local, RemoteEndpoint remote,
                      TransportProps transport, SecurityProps security);

        template<execution::queryable LocalEndpoint,
                 execution::queryable RemoteEndpoint,
                 execution::queryable TransportProps>
        preconnection(LocalEndpoint local, RemoteEndpoint remote,
                      TransportProps transport = /* 默认 */);

                 execution::queryable TransportProps>

                      TransportProps transport = /* 默认 */);
    };
}
```

### 6.17 解析端点

#### 6.17.1 `net::resolve`

```cpp
namespace std::net {
```
